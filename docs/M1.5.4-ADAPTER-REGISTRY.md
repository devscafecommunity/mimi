# M1.5.4: Adapter Registry & Discovery

## Overview

Production-grade adapter registry with health monitoring, configuration-driven selection, and automatic failover.

## Components

### AdapterHealth

Tracks health metrics for each adapter:
- **is_healthy**: Boolean flag (healthy if < 3 consecutive failures)
- **consecutive_failures**: Count since last success
- **success_count / error_count**: Lifetime stats
- **last_latency_ms**: Last recorded response time
- **success_rate()**: Calculate success percentage
- **average_latency_ms()**: Calculate avg latency
- **is_recently_checked()**: Check if within N seconds

### HealthChecker

Background task that periodically verifies adapter health:
- Spawns on factory creation
- Runs capability test on each adapter (send dummy request, measure latency)
- Updates health status in registry
- Configurable interval (default: 30 seconds)

### AdapterRegistry (Enhanced)

Core registry with health tracking and fallback logic:

#### Key Methods:
```rust
pub async fn register_with_health(&self, name: String, adapter: SharedAdapter) -> AdapterResult<()>
pub async fn get_best_available(&self) -> AdapterResult<SharedAdapter>
pub async fn get_health(&self, name: &str) -> AdapterResult<AdapterHealth>
pub async fn record_success(&self, name: &str, latency_ms: u32) -> AdapterResult<()>
pub async fn record_failure(&self, name: &str) -> AdapterResult<()>
pub async fn list_healthy(&self) -> Vec<String>
```

#### Fallback Logic:
1. Iterate through priority list (e.g., gemini → ollama)
2. Return first adapter with `is_healthy = true`
3. If none healthy, return first in priority order (fallback)
4. If none exist, return `AllAdaptersFailed` error

### AdapterPriority Config

Configuration for adapter selection:
```rust
pub struct AdapterPriority {
    pub adapters: Vec<String>,
    pub check_interval_secs: u64,
}
```

**Loading (Hybrid):**
1. Check for `adapters.toml` in current directory
2. If found and valid, use it
3. Otherwise, use hardcoded defaults: `["gemini", "ollama"]` with 30s interval

**Example adapters.toml:**
```toml
[adapters]
adapters = ["ollama", "gemini"]
check_interval_secs = 60
```

### AdapterFactory (Enhanced)

Factory with registry creation and config loading:

```rust
pub async fn create(config: &AdapterConfig) -> AdapterResult<SharedAdapter>
pub async fn create_with_registry(config: &AdapterConfig) -> AdapterResult<Arc<AdapterRegistry>>
async fn load_priority() -> AdapterResult<AdapterPriority>
```

## Data Flow

```
┌─────────────────────────────────────────┐
│ AdapterFactory::create_with_registry()  │
│  1. Load adapter priority               │
│  2. Create registry                     │
│  3. Create + register adapters          │
│  4. Spawn health checker                │
└─────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────┐
│ HealthChecker (background)              │
│  Loop every 30s:                        │
│   - Send capability test to each        │
│   - Update health status                │
└─────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────┐
│ Caller: registry.get_best_available()   │
│  1. Iterate priority list               │
│  2. Return first healthy                │
│  3. Fallback if none healthy            │
└─────────────────────────────────────────┘
```

## Error Handling

- Health check fails? → Mark unhealthy, don't crash
- No adapters available? → Return `AllAdaptersFailed` with details
- Config file missing? → Use hardcoded defaults
- Config file malformed? → Log warning, use defaults

## Usage Example

```rust
let config = AdapterConfig {
    adapter_type: "gemini".to_string(),
    api_key: Some(env::var("GEMINI_API_KEY").unwrap()),
    endpoint: Some("https://generativelanguage.googleapis.com".to_string()),
    timeout_ms: 30000,
    max_retries: 3,
    model: "gemini-pro".to_string(),
};

// Create registry with health checking
let registry = AdapterFactory::create_with_registry(&config).await?;

// Get best available adapter (respects priority + health)
let adapter = registry.get_best_available().await?;

// Use adapter...
let response = adapter.lock().await.invoke(&request).await?;

// Record success/failure after use
registry.record_success("gemini", 45).await?;
```

## Performance

- Health check: Non-blocking background task (30s interval default)
- Registry lookup: O(n) where n = number of adapters (~2-3 typically)
- Registry operations: RwLock-based, minimal contention
- Fallback chain: Checked in priority order
