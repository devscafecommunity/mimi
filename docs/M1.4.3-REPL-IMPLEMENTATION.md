## Overview

The M1.4.3 REPL (Read-Eval-Print Loop) provides an interactive shell interface for continuous user interaction with MiMi. This implementation delivers:

- **Command-line REPL** with full readline support for line editing
- **Special commands** (`.help`, `.status`, `.clear`, `.config`, `.history`, `.exit`, `.quit`)
- **Shell command execution** via `!command` syntax (uses fork + exec)
- **Multi-line input support** with backslash continuation (`\`)
- **Startup script execution** to run initialization commands
- **Command history** with configurable in-memory storage
- **Prompt states** for different contexts (default, following tasks, debug mode)

## Architecture

### Module Structure

```
src/repl/
├── mod.rs                  # Core configuration (ReplConfig, PromptState)
├── editor.rs              # History management (in-memory, Phase 1)
├── completion.rs          # Completion provider (stub for Phase 2)
├── special_commands.rs    # Special command parsing and execution
└── loop.rs               # Main REPL event loop and line processing

tests/
└── repl_integration_tests.rs  # 12 integration tests
```

### Data Flow

1. **Initialization**
   - Parse command-line arguments (--startup-script, --no-history, etc.)
   - Load startup script if provided
   - Initialize ReplEditor with configured history limit

2. **Main Loop**
   - Print prompt based on PromptState
   - Read line from stdin via BufRead
   - Check for line continuation (`\` suffix) → enter Continuation state
   - Buffer multiline input
   - Parse special commands (`.` prefix) or shell commands (`!` prefix)
   - Execute or defer to future integration

3. **Command Dispatch**
   - Special commands → immediate execution
   - Shell commands → fork subprocess via `std::process::Command`
   - Regular commands → buffered (deferred to M1.4.6)

4. **History Management**
   - Add each complete line to history
   - Maintain configurable max length (default 1000)
   - Optional persistence (implemented in Phase 2 with rustyline)

## API Reference

### ReplConfig

Configuration structure for REPL behavior:

```rust
pub struct ReplConfig {
    pub history_file: String,        // Path to history file
    pub max_history: usize,          // Max stored entries (default: 1000)
    pub no_history: bool,            // Disable history (default: false)
    pub startup_script: Option<String>, // Script to run on init
    pub completion: bool,            // Enable completion (default: true)
}
```

### PromptState

Enum representing different prompt contexts:

```rust
pub enum PromptState {
    Default,              // "mimi> "
    Following(u32),       // "mimi [task-123]> " (for async tasks)
    Debug,                // "mimi [debug]> "
    Continuation,         // "... " (for multiline input)
}
```

### SpecialCommand

Enum for dot-commands:

```rust
pub enum SpecialCommand {
    Help(Option<String>),           // .help [CMD]
    Status,                         // .status
    Clear,                          // .clear
    ConfigGet(String),              // .config get <KEY>
    ConfigSet(String, String),      // .config set <KEY> <VAL>
    History(Option<usize>),         // .history [N]
    Exit,                           // .exit / .quit
    Shell(String),                  // !command
}
```

### run_repl()

Main REPL entry point:

```rust
pub async fn run_repl(config: ReplConfig) -> io::Result<()>
```

## Command Reference

### Special Commands (dot-prefixed)

| Command | Syntax | Purpose | Example |
|---------|--------|---------|---------|
| Help | `.help [COMMAND]` | Show help | `.help exec` |
| Status | `.status` | System status | `.status` |
| Clear | `.clear` | Clear screen | `.clear` |
| Config Get | `.config get <KEY>` | Get config value | `.config get adapter.default` |
| Config Set | `.config set <KEY> <VAL>` | Set config value | `.config set debug.verbosity 2` |
| History | `.history [N]` | Show last N commands | `.history 20` |
| Exit/Quit | `.exit` or `.quit` | Exit REPL | `.exit` |

### Shell Commands (bang-prefixed)

Execute system shell commands:

```bash
!ls -la                 # Execute ls with args
!pwd                    # Show current directory
!grep pattern file.txt  # Run grep
```

### Regular Commands

User input without special prefixes is buffered for future command execution:

```
mimi> exec create skill for text processing
REPL: parsed command 'exec create skill for text processing' (execution deferred to M1.4.6)
```

## Usage Examples

### Basic REPL Usage

```bash
# Start REPL
$ mimi repl
mimi> .status
System status: OK
mimi> .help
Special REPL commands:
  .help [COMMAND]     - Show help
  .status             - Show system status
  ...
mimi> .exit
```

### Multi-line Input

```bash
mimi> exec create skill \
... for document \
... processing
REPL: parsed command 'exec create skill for document processing' (execution deferred to M1.4.6)
```

### Shell Integration

```bash
mimi> !echo current directory:
current directory:
mimi> !pwd
/home/user/project
mimi> .exit
```

### Startup Script

Create `startup.mimi`:
```
.status
```

Run with script:
```bash
$ mimi repl --startup-script startup.mimi
mimi> System status: OK
mimi> .exit
```

### Custom Configuration

```bash
# Disable history
$ mimi repl --no-history

# Custom history file
$ mimi repl --history-file ~/my-mimi-history

# Limit history size
$ mimi repl --max-history 500
```

## Implementation Details

### Multi-line Input Handling

Lines ending with backslash (`\`) enter continuation mode:

```rust
if line.ends_with('\\') {
    multiline_buffer.push_str(&line[..line.len() - 1]);
    multiline_buffer.push(' ');
    prompt_state = PromptState::Continuation;
    continue;
}
```

The continuation prompt (`"... "`) indicates readiness for more input.

### Special Command Execution

All special commands have immediate effects:

```rust
pub enum LineExecution {
    Exit,      // Break from loop
    Continue,  // Process next line
}
```

- `.exit` / `.quit` → `LineExecution::Exit`
- `.clear` → Print ANSI escape code, continue
- `.history` → Print history from editor, continue
- Others → Print output, continue

### Shell Command Safety

Shell commands executed via `std::process::Command`:

```rust
std::process::Command::new("sh")
    .arg("-c")
    .arg(cmd)
    .output()
```

- Uses `sh -c` for consistent behavior across platforms
- Captures stdout/stderr for display
- Returns `io::Result` for error handling

## Design Decisions

### Phase 1 vs Phase 2

**Phase 1 (This Release):**
- In-memory history only (simple Vec<String>)
- No readline/line editing features
- Basic stdin/stdout I/O

**Phase 2 (Future):**
- rustyline integration for persistent history
- Terminal line editing (arrow keys, Ctrl-A, etc.)
- Auto-completion of commands and arguments
- Syntax highlighting

### Why No Persistent History in Phase 1?

- Keep scope limited for faster delivery
- rustyline adds significant complexity
- In-memory history sufficient for initial use cases
- Phase 2 can add persistence without breaking Phase 1

### Command Parsing Strategy

Two-tier parsing:
1. **Special commands** (`.` prefix) → parse directly in REPL
2. **Regular commands** → buffer for later processing (M1.4.6)

This allows:
- REPL controls its own flow (exit, clear, etc.)
- Regular commands can be processed by Mimi state machine
- Clean separation of concerns
