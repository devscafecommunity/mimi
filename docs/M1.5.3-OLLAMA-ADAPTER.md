# M1.5.3 - Ollama AI Adapter

## Overview

The Ollama adapter enables mimi to integrate with Ollama for local and cloud-based language model inference. Ollama is a lightweight framework that simplifies running large language models locally while also supporting cloud-based deployments.

**Key Features:**
- Dual-mode operation: Local (http://localhost:11434) and Cloud (https://ollama.com)
- Automatic mode detection based on API key presence
- Full async/await support with tokio
- Thread-safe client management using Arc<Mutex<>>
- Lazy client initialization (created on first use)
- Comprehensive health checks for both modes
- Support for multiple Ollama models (llama2, codellama, mistral, neural-chat)

## Architecture

### Local Mode
Local Ollama instances run on the host machine and communicate via HTTP to the standard Ollama port (11434).

**Requirements:**
- Ollama installed and running locally
- No API key required
- Fast, low-latency inference

**Example Configuration:**
```rust
let config = AdapterConfig {
    adapter_type: "ollama".to_string(),
    api_key: None,  // No key needed for local
    model: "llama2".to_string(),
    timeout_ms: 5000,
    endpoint: Some("http://localhost:11434".to_string()),
    max_retries: 3,
};
```

### Cloud Mode
Cloud Ollama supports hosted model deployments on https://ollama.com.

**Requirements:**
- Active account on ollama.com
- Valid API key (obtain via `ollama signin`)
- API endpoint configured (typically https://ollama.com)

**Example Configuration:**
```rust
let config = AdapterConfig {
    adapter_type: "ollama".to_string(),
    api_key: Some("your-api-key".to_string()),  // Cloud auth
    model: "llama2".to_string(),
    timeout_ms: 5000,
    endpoint: Some("https://ollama.com".to_string()),
    max_retries: 3,
};
```

## Mode Selection Logic

The adapter automatically selects the appropriate mode:

| Condition | Mode | Description |
|-----------|------|-------------|
| `api_key.is_empty()` | Local | No API key = local Ollama instance |
| `api_key.is_not_empty()` | Cloud | API key provided = cloud authentication |

This allows flexible deployment: use local during development, switch to cloud for production by adding an API key.

## Client Implementation

### OllamaClient

Located in `crates/mimi-cli/src/ai/ollama_client.rs`

```rust
pub struct OllamaClient {
    client: Client,           // Reqwest HTTP client
    endpoint: String,         // Base URL for Ollama API
    api_key: Option<String>,  // Authentication (Cloud mode)
    mode: OllamaMode,        // Local or Cloud
    timeout: Duration,        // Request timeout
}
```

**Public Methods:**

1. **`new_local(endpoint: String, timeout_ms: u64) -> Result<Self>`**
   - Creates a client for local Ollama instance
   - No authentication required
   - Validates endpoint is not empty

2. **`new_cloud(api_key: String, endpoint: String, timeout_ms: u64) -> Result<Self>`**
   - Creates a client for cloud Ollama API
   - Requires non-empty API key
   - Sets up Bearer token authentication

3. **`invoke(request: OllamaRequest) -> Result<OllamaResponse>`**
   - Sends chat completion request to Ollama API
   - Uses `/api/chat` endpoint
   - Returns extracted text response
   - Handles authentication automatically based on mode

4. **`health_check() -> Result<()>`**
   - Pings Ollama `/api/tags` endpoint
   - Verifies connectivity and authentication
   - Used during initialization

### Error Handling

`OllamaClientError` variants:
- `RequestFailed` - HTTP communication error
- `ApiError` - Ollama API returned error
- `ParseError` - Failed to parse response JSON
- `Timeout` - Request exceeded timeout threshold
- `RateLimited` - API rate limit hit (HTTP 429)
- `InvalidRequest` - Invalid input parameters
- `AuthenticationFailed` - Invalid/missing credentials (HTTP 401/403)

## Adapter Implementation

Located in `crates/mimi-cli/src/ai/ollama_adapter.rs`

```rust
pub struct OllamaAdapter {
    client: Arc<Mutex<Option<OllamaClient>>>,  // Lazy initialization
    config: AdapterConfig,
    initialized: Arc<Mutex<bool>>,
}
```

**Implements `AiAdapter` trait:**

1. **`initialize(params: AdapterInitParams) -> Result<()>`**
   - Validates endpoint is provided
   - Determines mode based on api_key
   - Creates client (local or cloud)
   - Performs health check
   - Marks adapter as ready

2. **`invoke(request: AiRequest) -> Result<AiResponse>`**
   - Gets or creates client on first call
   - Uses configured model or request-provided model
   - Maps Ollama response to AiResponse
   - Sets `cached: false` (Ollama doesn't cache responses)

3. **`capabilities() -> Result<AdapterCapabilities>`**
   - Returns supported models: `[llama2, codellama, mistral, neural-chat]`
   - `supports_streaming: false` (implemented via non-streaming API)
   - `supports_caching: false` (no caching layer)
   - `max_context_tokens: 4096` (typical Ollama context window)

4. **`health_check() -> Result<()>`**
   - Delegates to client health check
   - Reuses or creates client

5. **`cleanup() -> Result<()>`**
   - Closes HTTP client
   - Resets initialized flag

6. **`adapter_name() -> String`**
   - Returns `"ollama"`

## Usage Example

### Setup Local Ollama

```bash
# Install Ollama
curl https://ollama.ai/install.sh | sh

# Start Ollama server
ollama serve

# In another terminal, pull a model
ollama pull llama2
```

### Use Adapter in mimi

```rust
use mimi_cli::ai::{AdapterFactory, AdapterConfig, AiAdapter, AiRequest};

#[tokio::main]
async fn main() -> Result<()> {
    // Configure adapter for local mode
    let config = AdapterConfig {
        adapter_type: "ollama".to_string(),
        api_key: None,
        model: "llama2".to_string(),
        timeout_ms: 30000,
        endpoint: Some("http://localhost:11434".to_string()),
        max_retries: 3,
    };

    // Create adapter
    let adapter = AdapterFactory::create(&config).await?;
    let adapter = adapter.lock().await;

    // Initialize
    let init_params = AdapterInitParams {
        api_key: String::new(),
        timeout_ms: 30000,
        endpoint: Some("http://localhost:11434".to_string()),
        max_retries: 3,
    };
    adapter.initialize(init_params).await?;

    // Make inference request
    let request = AiRequest {
        prompt: "What is machine learning?".to_string(),
        model: None,  // Use config model
        temperature: Some(0.7),
        max_tokens: Some(256),
        system_context: None,
    };

    let response = adapter.invoke(request).await?;
    println!("Response: {}", response.content);

    // Cleanup
    adapter.cleanup().await?;
    Ok(())
}
```

## Performance Characteristics

### Local Mode
- **Latency**: Sub-100ms for small models on modern hardware
- **Throughput**: Limited by hardware capabilities
- **Cost**: Free (only hardware cost)
- **Best For**: Development, local testing, privacy-sensitive workloads

### Cloud Mode
- **Latency**: 100-500ms depending on network and server load
- **Throughput**: Limited by API rate limits (typically 1-10 req/s)
- **Cost**: Pay-per-inference or subscription
- **Best For**: Production deployments, scalability, managed infrastructure

## Error Recovery

### Local Mode Issues
- **"Connection refused"**: Ensure Ollama is running (`ollama serve`)
- **"Model not found"**: Pull model with `ollama pull <model-name>`
- **Timeout**: Increase `timeout_ms` or check system resources

### Cloud Mode Issues
- **"Unauthorized (401)"**: Verify API key via `ollama signin` and update config
- **"Forbidden (403)"**: Check account permissions or rate limits
- **"Service unavailable (503)"**: Cloud service temporarily down, retry later

### General
- Adapter implements exponential backoff retry logic
- Check `max_retries` configuration if failures occur
- Enable logging for detailed error messages

## Migration from Other Adapters

If switching from Gemini to Ollama:

1. Replace adapter_type: `"gemini"` → `"ollama"`
2. Update endpoint configuration
3. Adjust model names (Gemini models not supported)
4. Remove unnecessary Gemini-specific config (e.g., cache settings)

**Before (Gemini):**
```rust
let config = AdapterConfig {
    adapter_type: "gemini".to_string(),
    api_key: Some("gemini-api-key".to_string()),
    model: "gemini-pro".to_string(),
    // ...
};
```

**After (Ollama):**
```rust
let config = AdapterConfig {
    adapter_type: "ollama".to_string(),
    api_key: None,  // Local mode
    model: "llama2".to_string(),
    endpoint: Some("http://localhost:11434".to_string()),
    // ...
};
```
