## Architecture & Code Quality

### Design Patterns

1. **LoadMetrics Struct**: Encapsulates percentile calculation and reporting
   - Automated p50/p95/p99 calculation from raw latency data
   - Formatted output with microsecond precision
   - Reusable across all test cases

2. **Test Organization**: Clear separation of 10 independent tests
   - Logical grouping with visual dividers
   - Assertion comments document performance targets
   - Each test isolated with its own auth manager instance

3. **Identity Creation**: Parameterized helper function
   - Reduces code duplication
   - Supports role variation for scaling tests

4. **Concurrent Testing**: Proper thread spawning and joining
   - Pre-generated tokens for validation tests
   - Clean latency collection across thread boundaries
   - Error handling for thread panics
