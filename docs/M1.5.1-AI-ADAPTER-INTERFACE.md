## Overview

M1.5.1 establishes the foundation for all AI integrations in MiMi. The interface uses Rust traits to define a contract that all AI providers (Gemini, Ollama, future providers) must implement.

## Architecture

### Components

1. **AiAdapter Trait** - Core abstraction
   - `initialize()` - Setup and authentication
   - `invoke()` - Synchronous AI model calls
   - `capabilities()` - Report adapter abilities
   - `health_check()` - Verify operation
   - `cleanup()` - Resource cleanup

2. **AdapterConfig** - Configuration management
   - Supports environment variables
   - Validates adapter-specific requirements
   - Provides sensible defaults

3. **AdapterFactory** - Adapter creation
   - Instantiates adapters by type
   - Maintains list of supported adapters
   - Extensible for new providers

4. **AdapterRegistry** - Adapter lifecycle
   - Register/unregister adapters
   - Query available adapters
   - Thread-safe access via RwLock

5. **Error Handling** - Unified error types
   - `AdapterError` enum with variants
   - `is_retryable()` for circuit breaker
   - Integration with anyhow

## Error Handling Strategy

All errors are retryable EXCEPT:
- InvalidRequest - client error, fix and retry
- ConfigError - fix configuration
- AdapterNotFound - adapter doesn't exist
- Unknown - depends on context

Retryable errors:
- Timeout
- RateLimited
- NetworkError

See `AdapterError::is_retryable()` for programmatic check.
