# M1.2.3: Zenoh Client Library Implementation

**Status:** Complete  
**Components:** Connection pooling, Pub/Sub, Request/Reply patterns

## Overview

This implementation provides a production-grade Zenoh client library wrapper for MiMi, abstracting low-level Zenoh APIs into ergonomic, type-safe interfaces with built-in error handling and async/await support.

## Architecture

### Modules

#### 1. `config` - Connection Configuration

```rust
pub struct ConnectionConfig {
    pub broker_endpoint: String,        // "tcp/127.0.0.1:7447"
    pub timeout_ms: u64,                // 5000ms default
    pub max_message_size: usize,        // 4MB default
    pub connection_pool_size: usize,    // 4 concurrent sessions
}
```

**Rationale:**
- Broker endpoint configurable for M1 (localhost) and M2+ (mesh)
- Message size limit aligns with FlatBuffers schema (4MB skill binaries)
- Pool size tuned for 10 MiMi modules on single machine

#### 2. `pool` - Connection Pooling

**Key Features:**
- Fixed-size session pool (default 4 sessions)
- Round-robin load distribution
- Automatic session initialization
- Graceful shutdown

**Implementation:**
```rust
pub struct ConnectionPool {
    sessions: Vec<Arc<Session>>,
    current_index: Arc<RwLock<usize>>,
}
```

**Usage:**
- Prevents connection storms (1 pool vs 10 individual sessions)
- Thread-safe round-robin: `*idx = (*idx + 1) % pool_size`
- Sessions wrapped in Arc<Session> for shared ownership

#### 3. `publisher` - Pub/Sub Publishing

**Capabilities:**
- Reliable & Best-Effort modes (QoS selection)
- Congestion control options
- Topic-based routing

**Example:**
```rust
let pub_reliable = client.publisher("mimi/task/execute", true);
pub_reliable.publish(b"binary_payload").await?;

let pub_best_effort = client.publisher("sys/telemetry", false);
pub_best_effort.publish_with_congestion_control(
    b"metrics",
    CongestionControl::Block
).await?;
```

#### 4. `subscriber` - Pub/Sub Subscription

**Capabilities:**
- Async message receiving
- Configurable buffer size
- Timeout support

**Example:**
```rust
let mut sub = client.subscriber("beatrice/intent", 100).await?;

match sub.recv_timeout(Duration::from_secs(5)).await {
    Some(msg) => process(msg),
    None => eprintln!("timeout"),
}
```

#### 5. `request_reply` - Synchronous RPC

**Components:**

**Replier (Server):**
```rust
let replier = client.replier("rpc/execute", |request| {
    execute_task(request).map(|result| result.serialize())
});
replier.start().await?;
```

**Requester (Client):**
```rust
let requester = client.requester("rpc/execute", Duration::from_secs(5));
let response = requester.request(b"task_data").await?;
```

#### 6. `client` - Unified Interface

Factory for all pub/sub and RPC operations:

```rust
let cfg = ConnectionConfig::default();
let client = ZenohClient::new(cfg).await?;

let pub_task = client.publisher("mimi/task/execute", true);
let mut sub_intent = client.subscriber("beatrice/intent", 10).await?;
let requester = client.requester("rpc/query", Duration::from_secs(5));
```

## Topic Hierarchy

Maps to BUS-PROTOCOL.md specifications:

| Topic | Pattern | QoS | Producer | Consumer |
|-------|---------|-----|----------|----------|
| `beatrice/input` | Pub/Sub | BestEffort | User → Beatrice | Mimi |
| `beatrice/intent` | Pub/Sub | BestEffort | Beatrice | Liliana, Mimi |
| `liliana/response` | Pub/Sub | Reliable | Liliana | Odlaguna |
| `mimi/task/execute` | Request/Reply | Reliable | Mimi | Ryzu |
| `mimi/task/result` | Pub/Sub | Reliable | Ryzu | Mimi |
| `pandora/memory/update` | Pub/Sub | Reliable | Mimi | Pandora |
| `odlaguna/gate` | Pub/Sub | Reliable | Odlaguna | Liliana |
| `sys/error` | Pub/Sub | BestEffort | Any | Odlaguna |
| `sys/telemetry` | Pub/Sub | BestEffort | Any | Prometheus |

## Error Handling

All operations return `Result<T, anyhow::Error>`:

```rust
match client.publisher("topic", true).publish(data).await {
    Ok(_) => println!("Published"),
    Err(e) => eprintln!("Publish failed: {}", e),
}
```

**Error Types:**
- `"Failed to open Zenoh session"` - Connection failure
- `"Publish failed"` - Topic misconfiguration or broker down
- `"Request timeout"` - Replier didn't respond in time
- `"No response received"` - Replier connection lost

## Integration with MiMi Modules

### Example: Beatrice → Mimi Pub/Sub

```rust
beatrice_module() {
    let client = ZenohClient::new(cfg).await?;
    let pub_intent = client.publisher("beatrice/intent", false);

    for user_input in parse_inputs() {
        let intent = classify(user_input);
        pub_intent.publish(&intent.serialize()).await?;
    }
}

mimi_module() {
    let client = ZenohClient::new(cfg).await?;
    let mut sub_intent = client.subscriber("beatrice/intent", 10).await?;

    while let Some(msg) = sub_intent.recv().await {
        let intent: IntentMessage = deserialize(&msg)?;
        route_intent(intent).await?;
    }
}
```

### Example: Mimi → Ryzu Request/Reply

```rust
mimi_module() {
    let requester = client.requester("mimi/task/execute", Duration::from_secs(5));
    let task = build_task(...);

    match requester.request(&task.serialize()).await {
        Ok(result) => handle_result(result),
        Err(_) => Odlaguna::report_failure(task),
    }
}

ryzu_module() {
    let replier = client.replier("mimi/task/execute", |req| {
        execute_skill(deserialize(&req)?)
    });
    replier.start().await?;
}
```
