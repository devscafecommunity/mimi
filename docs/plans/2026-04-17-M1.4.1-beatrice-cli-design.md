## Executive Summary

This document defines the complete CLI architecture for **Beatrice**, the natural language interface for the MiMi cognitive operating system. The design emphasizes:

- **Clear command hierarchy** with predictable patterns
- **Comprehensive help and error handling** for usability
- **Extensibility** for future adapter types and commands
- **Type-safe argument parsing** using Rust's `clap` framework
- **Consistency** with Unix CLI conventions

---

## 1. Command Structure

### Root Command: `mimi`

All commands follow the pattern:
```bash
mimi [GLOBAL_OPTIONS] <COMMAND> [COMMAND_OPTIONS] [ARGS]
```

#### Global Options (always available)
```
-v, --verbose              Enable verbose logging (repeatable: -vv, -vvv)
-q, --quiet                Suppress non-essential output
-c, --config <FILE>        Use alternate config file (default: ~/.config/mimi/config.toml)
--log-level <LEVEL>        Set log level: trace, debug, info, warn, error (default: info)
--output <FORMAT>          Output format: text, json, yaml (default: text)
--no-color                 Disable colored output
--help                     Show help message
--version                  Show version and build info
```

#### Exit Codes
```
0    Success
1    General error
2    Usage error (invalid arguments)
3    Configuration error (missing/invalid config)
4    Network error (bus/backend unavailable)
5    Authentication error
6    Not found error
64   I/O error
99   Unknown error
```

## 2. Command Hierarchy

### Level 1: Main Commands

```
mimi exec      Execute a task or script
mimi query     Ask a question or retrieve information
mimi config    Manage MiMi configuration
mimi debug     Internal debugging and diagnostics
mimi repl      Start interactive REPL mode (default if no command)
mimi version   Show version information
```

### 2.1 `mimi exec` — Execute Tasks

Execute structured actions within MiMi.

```bash
mimi exec [OPTIONS] <TASK> [TASK_ARGS...]

COMMAND OPTIONS:
  -p, --priority <PRIORITY>     Task priority: low, normal, high (default: normal)
  -t, --timeout <SECONDS>       Task timeout in seconds (default: 300)
  --async                       Run task asynchronously (return immediately with task ID)
  --follow                      Follow async task completion in real-time
  --save-result <FILE>          Save result to file
  --format <FORMAT>             Output format: text, json, yaml
  --dry-run                     Preview task without executing
```

#### Examples:
```bash
# Execute a simple task
mimi exec "create a new skill for PDF parsing"

# Execute with high priority
mimi exec -p high "urgent: fix the auth module"

# Run async and follow
mimi exec --async --follow "long running task"

# Save result to file
mimi exec "analyze codebase" --save-result result.json --format json

# Dry run preview
mimi exec --dry-run "implement feature X"
```

#### Sub-commands (future extensibility):
```
mimi exec create <TYPE>        Create new resources (skills, workflows, etc)
mimi exec run <NAME>           Run named skill/workflow
mimi exec stream <STREAM_ID>   Stream real-time output
```

### 2.2 `mimi query` — Query Information

Retrieve information from MiMi's memory and current state.

```bash
mimi query [OPTIONS] <QUERY_TEXT>

COMMAND OPTIONS:
  -f, --filter <FILTER>         Filter results by criteria (supports JSON jq syntax)
  -s, --sort <FIELD>            Sort results by field
  --limit <NUM>                 Limit results (default: 20)
  --offset <NUM>                Pagination offset (default: 0)
  --include-context             Include contextual information
  --memory-type <TYPE>          Search in: short, long, working, all (default: all)
  --from <DATE>                 Results from date (ISO 8601)
  --to <DATE>                   Results until date (ISO 8601)
  --format <FORMAT>             Output format: text, json, yaml
```

#### Examples:
```bash
# Simple query
mimi query "What's the status of the project?"

# Query with filtering
mimi query "list all failed tasks" -f "status == 'failed'"

# Query with pagination
mimi query "show recent interactions" --limit 50 --offset 0

# Query with date range
mimi query "tasks from last week" --from 2026-04-10 --to 2026-04-17

# Query with context
mimi query "explain the decision" --include-context --format json
```

#### Sub-commands:
```
mimi query memory               Query entire memory graph
mimi query skills              List available skills
mimi query status              Show MiMi current state
mimi query config              Show current configuration
mimi query logs <COMPONENT>    Query component logs (mimi, beatrice, pandora, etc)
```

### 2.3 `mimi config` — Configuration Management

Manage MiMi settings, adapters, and environment.

```bash
mimi config [SUBCOMMAND] [OPTIONS]

SUBCOMMANDS:
  set <KEY> <VALUE>             Set config value
  get <KEY>                     Get config value
  list                          List all config values
  reset                         Reset to defaults (with confirmation)
  validate                      Validate config file
  init <PROFILE>                Initialize with preset profile
```

#### Examples:
```bash
# Set values
mimi config set adapter.default gemini
mimi config set log.level debug
mimi config set memory.max_size 1gb

# Get values
mimi config get adapter.default
mimi config get memory.max_size

# List all
mimi config list

# Validate
mimi config validate

# Initialize with profile
mimi config init development   # Sets dev-friendly defaults
mimi config init production    # Sets production-safe defaults
```

#### Config Keys (Hierarchical):
```
adapter.default              Current AI adapter (gemini, ollama, claude)
adapter.gemini.api_key      (env override: MIMI_ADAPTER_GEMINI_API_KEY)
adapter.gemini.model        (default: gemini-pro)
adapter.ollama.url          (default: http://localhost:11434)

bus.url                      Message bus URL
bus.timeout_ms              (default: 5000)

log.level                   (trace, debug, info, warn, error)
log.format                  (text, json)

memory.max_size             Max memory graph size
memory.cleanup_interval     Cleanup interval in seconds

security.enable_audit       Enable security audit logging
security.timeout_per_task   Task execution timeout
```

### 2.4 `mimi debug` — Debugging & Diagnostics

Internal debugging tools for developers and troubleshooting.

```bash
mimi debug [SUBCOMMAND] [OPTIONS]

SUBCOMMANDS:
  status                        Show system status and health
  bus-latency <NUM_MESSAGES>    Test message bus latency
  memory-stats                  Show memory graph statistics
  adapter-test <ADAPTER>        Test AI adapter connectivity
  trace-flow <TASK_ID>          Trace execution flow of task
  profile <DURATION_SECS>       CPU/memory profiling
  validate-intent <TEXT>        Test intent parsing
```

#### Examples:
```bash
# System status
mimi debug status

# Test bus latency (send 1000 messages, report latency stats)
mimi debug bus-latency 1000

# Test adapter
mimi debug adapter-test gemini  # Requires MIMI_ADAPTER_GEMINI_API_KEY

# Trace a task
mimi debug trace-flow task-12345

# Profile CPU/memory for 30 seconds
mimi debug profile 30
```

### 2.5 `mimi repl` — Interactive REPL Mode

Start an interactive read-eval-print loop for continuous interaction.

```bash
mimi repl [OPTIONS]

COMMAND OPTIONS:
  --history-file <FILE>         History file location (default: ~/.mimi/repl_history)
  --max-history <NUM>           Max history entries (default: 1000)
  --no-history                  Disable history persistence
  --startup-script <FILE>       Run commands from file on startup
  --completion                  Enable command completion
```

#### REPL Prompt Behavior:
```
mimi> _                          # Ready for input
mimi [task-123]> _              # Following async task
mimi [debug]> _                 # In debug mode

# Multi-line input:
mimi> exec create skill \
... for document processing \
... with PDF support
... EOF or ^D to submit
```

#### REPL Special Commands:
```
.help [COMMAND]                 Show help
.status                         Show system status
.clear                          Clear screen
.config [get|set] <KEY> [VAL]   Quick config access
.history [N]                    Show last N commands (default: 20)
.exit / .quit / ^D              Exit REPL
!<SHELL_CMD>                    Run shell command (e.g. !ls)
```

#### Examples:
```bash
# Start REPL
mimi repl

# Start with custom history
mimi repl --history-file ~/project-mimi-history

# Start with startup script
mimi repl --startup-script setup.mimi

# Inside REPL:
mimi> query what's the current status
mimi> exec create skill for JSON validation
mimi> .config get adapter.default
mimi> .history 10
mimi> .exit
```

## 4. Error Handling & User Experience

### Error Message Format

```
ERROR [CODE]: Brief description
  cause: Why this happened
  hint:  What to try next
  docs:  https://docs.mimi.dev/errors#[CODE]
```

#### Example:
```
ERROR [E_CONFIG_NOT_FOUND]: Configuration file not found
  path: ~/.config/mimi/config.toml
  hint: Run 'mimi config init production' to create default config
  docs: https://docs.mimi.dev/errors#E_CONFIG_NOT_FOUND
```

### Validation Error Messages

For invalid arguments:
```
ERROR: Invalid argument 'invalid_priority'
  --priority must be one of: low, normal, high
  Current: --priority invalid_priority
  Try: mimi exec -p normal <task>
```

### Network Error Messages

```
ERROR [E_BUS_UNAVAILABLE]: Cannot connect to message bus
  address: localhost:7447
  hint: Ensure Zenoh bus is running: docker-compose up bus
  docs: https://docs.mimi.dev/errors#E_BUS_UNAVAILABLE
```

## 7. Command Line Parsing Strategy (clap)

### Using `clap` with Derive Macros

```rust
use clap::{Parser, Subcommand, ValueEnum};

#[derive(Parser)]
#[command(name = "mimi")]
#[command(version = "0.1.0")]
#[command(about = "Multimodal Instruction Master Interface", long_about = None)]
struct Cli {
    #[command(flatten)]
    global_opts: GlobalOpts,

    #[command(subcommand)]
    command: Commands,
}

#[derive(Parser)]
struct GlobalOpts {
    #[arg(short, long, action = clap::ArgAction::Count)]
    verbose: u8,

    #[arg(short, long)]
    config: Option<std::path::PathBuf>,

    #[arg(long, value_enum, default_value = "text")]
    output: OutputFormat,

    #[arg(long)]
    no_color: bool,
}

#[derive(ValueEnum, Clone, Debug)]
enum OutputFormat {
    Text,
    Json,
    Yaml,
}

#[derive(Subcommand)]
enum Commands {
    Exec {
        #[arg(value_name = "TASK")]
        task: String,

        #[arg(short, long, value_enum, default_value = "normal")]
        priority: Priority,

        #[arg(short, long, default_value = "300")]
        timeout: u32,

        #[arg(long)]
        async_mode: bool,
    },
    Query {
        #[arg(value_name = "QUERY")]
        query: String,

        #[arg(short, long)]
        filter: Option<String>,
    },
    Config { /* ... */ },
    Debug { /* ... */ },
    Repl { /* ... */ },
}
```

## 8. Exit Codes Summary

| Code | Meaning | Example |
|------|---------|---------|
| 0 | Success | Command executed successfully |
| 1 | General Error | Task failed, no specific category |
| 2 | Usage Error | Invalid arguments: `--priority INVALID` |
| 3 | Config Error | Config file missing or invalid |
| 4 | Network Error | Bus unreachable |
| 5 | Auth Error | API key missing/invalid |
| 6 | Not Found | Resource doesn't exist |
| 64 | I/O Error | File read/write error |
| 99 | Unknown Error | Unexpected exception |

## 9. Future Extensibility

This design allows for future additions without breaking existing commands:

- **New Commands**: Add to `Commands` enum, no changes to existing handlers
- **New Adapters**: Add to config, no CLI changes needed
- **New Output Formats**: Extend `OutputFormat` enum, add formatter implementations
- **Plugins**: Support loading external commands via WASM (future)

## 10. Design Decisions & Rationale

| Decision | Rationale |
|----------|-----------|
| `clap` derive macros | Type-safe, minimal boilerplate, excellent error messages |
| Global options before subcommand | Unix convention, better UX than after |
| Hierarchical config keys (dot notation) | Scales well for complex settings, familiar from many tools |
| JSON as first-class output | Essential for scripting and integration |
| Help text inlined in code | Single source of truth, doesn't drift from implementation |
| Exit codes follow sysexits.h | Standard across Unix tools, easy to script |
