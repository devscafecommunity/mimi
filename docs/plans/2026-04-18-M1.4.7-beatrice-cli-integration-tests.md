## Objective

Create comprehensive integration tests for CLI command workflows using the complete M1.4 stack:
- All CLI commands (exec, query, config, debug, repl, version)
- HTTP backend integration
- WebSocket backend integration
- Authentication flows (token validation, role-based access)
- Error handling and exit codes

## Test Architecture

### Test Environment
- **Scope**: Full end-to-end CLI + backend integration
- **Backend**: Local HTTP server with WebSocket support (spawned per test or shared)
- **Auth**: JWT token generation + validation (via M1.4.6 AuthManager)
- **Isolation**: Each test suite spawns CLI subprocesses + backends as needed
- **Teardown**: Cleanup servers and temp files after each test

### Test Categories

1. **CLI Workflow Tests**
   - Argument parsing and validation for each command type
   - Config file loading and merging
   - Output formatting (JSON, table, raw)
   - Logging levels (debug, info, warn, error)

2. **Backend Integration Tests**
   - CLI + HTTP server communication (exec, query, config)
   - CLI + WebSocket server communication (streaming results)
   - Connection pooling and reuse
   - Error propagation from backend to CLI

3. **Authentication Tests**
   - Token generation and embedding (--token flag)
   - Environment variable token loading (MIMI_AUTH_TOKEN)
   - Token validation and rejection (expired, invalid, missing)
   - Role-based access control (admin commands vs user queries)

4. **Error Handling Tests**
   - Invalid command arguments
   - Network errors (connection refused, timeout)
   - Auth errors (forbidden, unauthorized)
   - Backend errors (invalid query, command failure)
   - Exit codes validation (0 for success, 1 for error, etc.)

## Implementation Plan

### Task 1: CLI Integration Test Structure & Utilities
**Goal**: Set up test helpers for spawning CLI, servers, and managing test lifecycle

**Subtasks**:
- Create `/crates/mimi-cli/tests/cli_integration_utils.rs`:
  - `CliTestServer` struct - manages HTTP + WebSocket servers
  - `CliTestRunner` struct - spawns CLI subprocess, captures output
  - `TestContext` struct - holds both servers and CLI config
  - `spawn_test_servers()` - creates isolated HTTP/WS servers on random ports
  - `run_cli_command()` - executes CLI with args, captures stdout/stderr/exit_code
  - `cleanup()` - stops servers, removes temp files

- Create helper functions:
  - `create_test_config()` - generates temp config file
  - `create_test_token(identity)` - generates JWT token for testing
  - `assert_cli_success(output)` - validates exit code 0
  - `assert_cli_error(output, expected_code)` - validates error exit codes

**Dependencies**: 
- `std::process::{Command, Stdio}`
- `tokio::task` (for server spawning)
- Reuse `AuthManager` from M1.4.6

**Verification**:
- Utils compile without errors
- Helper functions work with mock CLI invocation
