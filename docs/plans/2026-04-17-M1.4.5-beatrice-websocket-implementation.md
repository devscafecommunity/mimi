# M1.4.5: Beatrice WebSocket Server Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Implement a WebSocket server with endpoints for streaming task execution and real-time query results, built on top of the existing HTTP infrastructure using actix-web's WebSocket support.

**Architecture:** WebSocket module added to `mimi-cli` crate alongside existing HTTP module. Extends Beatrice's CommandHandler pattern to support streaming responses via WebSocket connections. Clients can subscribe to `/ws/execute` (for streaming task execution output) and `/ws/query` (for real-time query results). Each connection maintains its own request context and streams JSON-formatted events to the client. WebSocket server shares the same port as HTTP (upgrades connections with HTTP Upgrade protocol).

**Tech Stack:** 
- Framework: actix-web 4.x (WebSocket support built-in)
- Protocol: JSON over WebSocket with event streaming
- Request parsing: serde + serde_json
- Error handling: thiserror + existing CliError enum
- Testing: actix-web WebSocket test helpers
- Config: Load WebSocket settings from TOML config (M1.4.2)

use serde::{Deserialize, Serialize};

/// WebSocket message sent by client
#[derive(Debug, Deserialize, Clone)]
pub struct WsMessage {
    /// Message ID for correlation
    pub id: String,
    /// Message type: "subscribe", "unsubscribe", "ping"
    pub msg_type: String,
    /// Subscription channel: "execute", "query"
    pub channel: String,
    /// Request payload (task, query, etc.)
    pub payload: serde_json::Value,
}

/// Event streamed from server to client
#[derive(Debug, Serialize, Clone)]
pub struct WsEvent {
    /// Message ID being responded to
    pub id: String,
    /// Event type: "start", "progress", "result", "error", "complete"
    pub event_type: String,
    /// Channel the event came from
    pub channel: String,
    /// Event timestamp (ISO 8601)
    pub timestamp: String,
    /// Event data
    pub data: serde_json::Value,
}

impl WsEvent {
    /// Create a new WebSocket event
    pub fn new(
        id: String,
        event_type: String,
        channel: String,
        data: serde_json::Value,
    ) -> Self {
        WsEvent {
            id,
            event_type,
            channel,
            timestamp: chrono::Utc::now().to_rfc3339(),
            data,
        }
    }

    /// Create a "start" event
    pub fn start(id: String, channel: String, description: String) -> Self {
        Self::new(
            id,
            "start".to_string(),
            channel,
            serde_json::json!({
                "message": description,
                "status": "started"
            }),
        )
    }

    /// Create a "progress" event
    pub fn progress(id: String, channel: String, percent: u32, message: String) -> Self {
        Self::new(
            id,
            "progress".to_string(),
            channel,
            serde_json::json!({
                "percent": percent,
                "message": message
            }),
        )
    }

    /// Create a "result" event
    pub fn result(id: String, channel: String, result: serde_json::Value) -> Self {
        Self::new(
            id,
            "result".to_string(),
            channel,
            serde_json::json!({
                "result": result
            }),
        )
    }

    /// Create an "error" event
    pub fn error(id: String, channel: String, error_message: String) -> Self {
        Self::new(
            id,
            "error".to_string(),
            channel,
            serde_json::json!({
                "error_code": "WS_ERROR",
                "error_message": error_message
            }),
        )
    }

    /// Create a "complete" event
    pub fn complete(id: String, channel: String, details: serde_json::Value) -> Self {
        Self::new(
            id,
            "complete".to_string(),
            channel,
            serde_json::json!({
                "message": "Operation completed",
                "details": details
            }),
        )
    }
}

/// Status of a WebSocket connection
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum ConnectionStatus {
    Connected,
    Authenticated,
    Subscribed,
    Error,
    Closed,
}

/// WebSocket connection context
#[derive(Debug, Clone)]
pub struct ConnectionContext {
    pub connection_id: String,
    pub status: ConnectionStatus,
    pub subscribed_channels: Vec<String>,
}

use actix_web::ws;
use serde_json::json;
use std::fmt;

use crate::cli::error::CliError;

/// WebSocket-specific error type
#[derive(Debug)]
pub enum WsError {
    CliError(CliError),
    InvalidMessage(String),
    NotSubscribed(String),
    InvalidChannel(String),
    ConnectionClosed,
}

impl fmt::Display for WsError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            WsError::CliError(e) => write!(f, "{}", e.user_message()),
            WsError::InvalidMessage(msg) => write!(f, "Invalid message: {}", msg),
            WsError::NotSubscribed(channel) => write!(f, "Not subscribed to channel: {}", channel),
            WsError::InvalidChannel(channel) => write!(f, "Invalid channel: {}", channel),
            WsError::ConnectionClosed => write!(f, "WebSocket connection closed"),
        }
    }
}

impl From<CliError> for WsError {
    fn from(err: CliError) -> Self {
        WsError::CliError(err)
    }
}

impl From<WsError> for ws::Message {
    fn from(err: WsError) -> Self {
        let error_response = json!({
            "event_type": "error",
            "error_message": err.to_string()
        });
        ws::Message::Text(error_response.to_string())
    }
}

use actix_web::{web, HttpRequest, HttpResponse};
use actix_web_actors::ws;
use actix::Actor;
use uuid::Uuid;
use serde_json::json;

use super::types::{WsMessage, WsEvent, ConnectionContext, ConnectionStatus};
use super::error::WsError;
use crate::cli::handler::CommandHandler;
use crate::cli::commands::Commands;

/// WebSocket connection actor
pub struct WsConnection {
    pub handler: CommandHandler,
    pub context: ConnectionContext,
}

impl WsConnection {
    /// Create new WebSocket connection
    pub fn new(handler: CommandHandler) -> Self {
        WsConnection {
            handler,
            context: ConnectionContext {
                connection_id: Uuid::new_v4().to_string(),
                status: ConnectionStatus::Connected,
                subscribed_channels: vec![],
            },
        }
    }

    /// Handle incoming WebSocket message
    async fn handle_message(
        &mut self,
        msg: WsMessage,
    ) -> Result<WsEvent, WsError> {
        match msg.msg_type.as_str() {
            "subscribe" => self.handle_subscribe(&msg),
            "unsubscribe" => self.handle_unsubscribe(&msg),
            "ping" => Ok(WsEvent::new(
                msg.id.clone(),
                "pong".to_string(),
                msg.channel.clone(),
                json!({"status": "connected"}),
            )),
            _ => Err(WsError::InvalidMessage(format!(
                "Unknown message type: {}",
                msg.msg_type
            ))),
        }
    }

    /// Handle subscription request
    fn handle_subscribe(&mut self, msg: &WsMessage) -> Result<WsEvent, WsError> {
        // Validate channel name
        match msg.channel.as_str() {
            "execute" | "query" => {
                if !self.context.subscribed_channels.contains(&msg.channel) {
                    self.context.subscribed_channels.push(msg.channel.clone());
                }
                Ok(WsEvent::start(
                    msg.id.clone(),
                    msg.channel.clone(),
                    format!("Subscribed to {}", msg.channel),
                ))
            }
            _ => Err(WsError::InvalidChannel(msg.channel.clone())),
        }
    }

    /// Handle unsubscription request
    fn handle_unsubscribe(&mut self, msg: &WsMessage) -> Result<WsEvent, WsError> {
        self.context
            .subscribed_channels
            .retain(|c| c != &msg.channel);
        Ok(WsEvent::new(
            msg.id.clone(),
            "unsubscribed".to_string(),
            msg.channel.clone(),
            json!({"message": "Unsubscribed from channel"}),
        ))
    }
}

impl Actor for WsConnection {
    type Context = ws::WebsocketContext<Self>;
}

/// WebSocket endpoint: /ws/execute - Stream task execution
pub async fn ws_execute_handler(
    req: HttpRequest,
    stream: web::Payload,
    handler: web::Data<CommandHandler>,
) -> Result<HttpResponse, actix_web::Error> {
    let connection = WsConnection::new(handler.as_ref().clone());
    ws::start(connection, &req, stream)
}

/// WebSocket endpoint: /ws/query - Stream query results
pub async fn ws_query_handler(
    req: HttpRequest,
    stream: web::Payload,
    handler: web::Data<CommandHandler>,
) -> Result<HttpResponse, actix_web::Error> {
    let connection = WsConnection::new(handler.as_ref().clone());
    ws::start(connection, &req, stream)
}

/// WebSocket endpoint: /ws/health - Health check over WebSocket
pub async fn ws_health_handler(
    req: HttpRequest,
    stream: web::Payload,
) -> Result<HttpResponse, actix_web::Error> {
    let connection = WsConnection::new(CommandHandler::new(
        Default::default(),
        Default::default(),
    ));
    ws::start(connection, &req, stream)
}

use serde_json::Value;

use super::error::WsError;
use super::types::WsMessage;

/// Validate WebSocket execute request
pub fn validate_ws_execute_request(msg: &WsMessage) -> Result<(), WsError> {
    // Extract task from payload
    let task = msg
        .payload
        .get("task")
        .and_then(Value::as_str)
        .unwrap_or("");

    if task.is_empty() {
        return Err(WsError::InvalidMessage(
            "task field is required and must be non-empty".to_string(),
        ));
    }

    if task.len() > 10_000 {
        return Err(WsError::InvalidMessage(
            "task must be less than 10,000 characters".to_string(),
        ));
    }

    // Validate priority if provided
    if let Some(priority) = msg.payload.get("priority").and_then(Value::as_str) {
        if !["low", "normal", "high"].contains(&priority) {
            return Err(WsError::InvalidMessage(
                "priority must be one of: low, normal, high".to_string(),
            ));
        }
    }

    // Validate timeout if provided
    if let Some(timeout) = msg.payload.get("timeout").and_then(Value::as_u64) {
        if timeout < 1 || timeout > 3600 {
            return Err(WsError::InvalidMessage(
                "timeout must be between 1 and 3600 seconds".to_string(),
            ));
        }
    }

    // Validate format if provided
    if let Some(format) = msg.payload.get("format").and_then(Value::as_str) {
        if !["text", "json", "yaml"].contains(&format) {
            return Err(WsError::InvalidMessage(
                "format must be one of: text, json, yaml".to_string(),
            ));
        }
    }

    Ok(())
}

/// Validate WebSocket query request
pub fn validate_ws_query_request(msg: &WsMessage) -> Result<(), WsError> {
    // Extract query from payload
    let query = msg
        .payload
        .get("query")
        .and_then(Value::as_str)
        .unwrap_or("");

    if query.is_empty() {
        return Err(WsError::InvalidMessage(
            "query field is required and must be non-empty".to_string(),
        ));
    }

    if query.len() > 10_000 {
        return Err(WsError::InvalidMessage(
            "query must be less than 10,000 characters".to_string(),
        ));
    }

    // Validate limit if provided
    if let Some(limit) = msg.payload.get("limit").and_then(Value::as_u64) {
        if limit < 1 || limit > 1000 {
            return Err(WsError::InvalidMessage(
                "limit must be between 1 and 1000".to_string(),
            ));
        }
    }

    // Validate format if provided
    if let Some(format) = msg.payload.get("format").and_then(Value::as_str) {
        if !["text", "json", "yaml"].contains(&format) {
            return Err(WsError::InvalidMessage(
                "format must be one of: text, json, yaml".to_string(),
            ));
        }
    }

    Ok(())
}

/// Generate a stream of progress events for long-running operations
pub fn create_progress_stream(
    message_id: String,
    channel: String,
    num_steps: u32,
) -> Vec<WsEvent> {
    let mut events = Vec::new();

    // Start event
    events.push(WsEvent::start(
        message_id.clone(),
        channel.clone(),
        "Starting operation".to_string(),
    ));

    // Progress events (simplified - in real implementation these would be driven by task execution)
    for step in 1..=num_steps {
        let percent = (step * 100) / num_steps;
        events.push(WsEvent::progress(
            message_id.clone(),
            channel.clone(),
            percent,
            format!("Step {}/{}", step, num_steps),
        ));
    }

    events
}

# M1.4.5: WebSocket Server Implementation - API Documentation

## Overview

The WebSocket server provides real-time bidirectional communication for:
- **Streaming task execution** with progress updates
- **Live query results** with event-based updates
- **Health monitoring** over persistent connections

## Architecture

WebSocket runs on the same port as the HTTP server (default: 3000). Clients connect via `ws://localhost:3000/ws/*` and receive JSON-encoded events.

## Endpoints

### `/ws/execute` - Stream Task Execution

**Connection:** `ws://localhost:3000/ws/execute`

**Message Format (Client → Server):**
```json
{
  "id": "msg-123",
  "msg_type": "subscribe",
  "channel": "execute",
  "payload": {
    "task": "create a new PDF parser",
    "priority": "normal",
    "timeout": 300,
    "format": "json"
  }
}
```

**Event Format (Server → Client):**
```json
{
  "id": "msg-123",
  "event_type": "start",
  "channel": "execute",
  "timestamp": "2026-04-18T10:30:00Z",
  "data": {
    "message": "Starting task execution",
    "status": "started"
  }
}
```

**Event Types:**
- `start` - Task execution started
- `progress` - Execution progress update (includes percent and message)
- `result` - Execution result available
- `error` - Execution error occurred
- `complete` - Execution finished

**Example Progress Event:**
```json
{
  "id": "msg-123",
  "event_type": "progress",
  "channel": "execute",
  "timestamp": "2026-04-18T10:30:05Z",
  "data": {
    "percent": 50,
    "message": "Processing step 2/4"
  }
}
```

### `/ws/query` - Stream Query Results

**Connection:** `ws://localhost:3000/ws/query`

**Message Format (Client → Server):**
```json
{
  "id": "msg-456",
  "msg_type": "subscribe",
  "channel": "query",
  "payload": {
    "query": "what are the available skills?",
    "limit": 10,
    "format": "json"
  }
}
```

**Event Types:**
- `start` - Query processing started
- `progress` - Query processing progress
- `result` - Result batch received (streams multiple result events)
- `error` - Query error
- `complete` - All results delivered

### `/ws/health` - Health Check over WebSocket

**Connection:** `ws://localhost:3000/ws/health`

**Message Format:**
```json
{
  "id": "health-1",
  "msg_type": "ping",
  "channel": "health",
  "payload": {}
}
```

**Response (pong):**
```json
{
  "id": "health-1",
  "event_type": "pong",
  "channel": "health",
  "timestamp": "2026-04-18T10:30:00Z",
  "data": {
    "status": "connected"
  }
}
```

## Validation Rules

### Execute Request
- `task`: Required, 1-10,000 characters
- `priority`: Optional, one of: low, normal, high (default: normal)
- `timeout`: Optional, 1-3600 seconds (default: 300)
- `format`: Optional, one of: text, json, yaml (default: json)

### Query Request
- `query`: Required, 1-10,000 characters
- `limit`: Optional, 1-1000 (default: 20)
- `format`: Optional, one of: text, json, yaml (default: json)

{

  "channel": "execute",
  "timestamp": "2026-04-18T10:30:00Z",
  "data": {
