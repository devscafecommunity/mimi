## Objective

Establish performance baselines, conduct load testing, and optimize the CLI + backend stack for production readiness:
- Throughput benchmarks (requests/sec)
- Latency measurement (p50, p95, p99)
- WebSocket connection scaling
- Memory profiling under load
- Token generation performance
- Auth validation efficiency

## Performance Testing Architecture

### Metrics to Capture

1. **CLI Performance**
   - Command execution time (version, run, help)
   - Startup time
   - Memory footprint
   - Config file parsing time

2. **Auth Performance**
   - Token generation time (ms)
   - Token validation time (ms)
   - Permission checking time (ms)
   - Memory usage for policy storage

3. **Backend Performance**
   - HTTP request throughput (req/sec)
   - WebSocket message throughput (msg/sec)
   - Latency distribution (p50, p95, p99)
   - Connection pool efficiency

4. **Load Characteristics**
   - Peak throughput (concurrent requests)
   - Sustained throughput (sustained load)
   - Failure modes (overload behavior)
   - Resource exhaustion patterns

### Task 2: Load Testing & Stress Tests
**Goal**: Test behavior under sustained and peak load

**Test cases** (in `/crates/mimi-cli/tests/cli_load_tests.rs`):

1. `bench_token_generation_throughput` - Generate 1000 tokens, measure rate
2. `bench_token_validation_throughput` - Validate 1000 tokens, measure rate
3. `bench_permission_checks_throughput` - Check 10000 permissions, measure rate
4. `bench_auth_manager_scaling` - Register 100 policies, measure lookup time
5. `bench_cli_command_execution_batch` - Execute 100 commands sequentially
6. `bench_concurrent_token_generation` - Generate tokens from 10 concurrent threads
7. `bench_concurrent_validation` - Validate tokens from 10 concurrent threads
8. `bench_large_identity_lookup` - Search 1000 identities for permission
9. `bench_auth_policy_registry_size` - Measure auth manager memory with 1000 policies
10. `bench_cli_startup_overhead` - Measure CLI process spawn time (10 iterations)

**Metrics Collected**:
- Throughput (ops/sec)
- Latency (min, max, avg, p95, p99)
- Memory usage (peak, average)
- CPU time (user, system)

**Verification**:
- All load tests complete without resource exhaustion
- Latency p99 < 10ms for auth operations
- Throughput > 1000 ops/sec for token validation
- Memory stable (no leaks detected)

### Task 3: Memory Profiling & Optimization
**Goal**: Profile memory usage and identify optimization opportunities

**Profiling approach**:
- Use `valgrind` or similar for memory leak detection
- Track heap allocations during token generation
- Measure policy storage memory efficiency
- Profile CLI startup memory

**Optimization targets** (in `/crates/mimi-cli/src/`):
- Token serialization: Use `serde_json` compact format
- Policy storage: Consider string interning for policy names
- Identity lookup: Profile HashMap vs other structures
- JWT claims: Minimize claim size in tokens

**Verification**:
- No memory leaks detected in auth module
- Peak memory < 10MB for 1000 identities
- Memory stable after warm-up period
