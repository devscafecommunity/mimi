**Goal:** Validate full M1 flow: CLI command → Message Bus → Mimi core → AI adapter → response, plus HTTP/WebSocket/error paths.

**Architecture:** Create integration tests covering 4 major flows:
1. CLI workflow: CLI command parses → sends to Mimi via message bus → receives response
2. HTTP flow: HTTP request → message bus → Mimi → adapter → JSON response
3. WebSocket flow: WS connection → Mimi processes message → streams response back
4. Error scenarios: network failure, timeout, invalid input, adapter unavailable

Each test will start Mimi core, Beatrice (HTTP+WS), configure AI adapters, and verify end-to-end response.

**Tech Stack:** Tokio, reqwest (HTTP client), tungstenite (WebSocket client), serde_json, tempfile (config management)
