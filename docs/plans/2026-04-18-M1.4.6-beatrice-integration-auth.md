# M1.4.6: Beatrice Integration & Authentication Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Implement a unified authentication and authorization system that works across CLI, HTTP, and WebSocket interfaces, enabling secure multi-client access with role-based access control (RBAC).

**Architecture:** Authentication module (`mimi-cli/src/auth/`) provides token-based auth with JWT tokens, session management, and RBAC. Middleware components integrate with all three interfaces:
- CLI: Optional token authentication via `--token` flag or environment variable
- HTTP: Bearer token validation in `Authorization: Bearer <token>` header
- WebSocket: Token passed in initial handshake or via first message, validated on connect

All interfaces share a common `AuthMiddleware` that validates tokens, extracts identity/roles, and enforces RBAC policies. Routes/handlers check required roles before processing requests.

**Tech Stack:**
- Framework: actix-web middleware system
- Token generation: jsonwebtoken (JWT with HS256)
- Password hashing: argon2 (when user credentials involved)
- Session storage: In-memory (HashMap + Arc<Mutex>) for single-instance deployment
- Config: Auth settings from TOML (secret key, token TTL, roles, policies)
- Testing: Integration tests for all three interfaces with auth flows

pub enum Role {

    User,
    Guest,

impl Role {
    pub fn as_str(&self) -> &str {
        match self {
            Role::Admin => "admin",
            Role::User => "user",
            Role::Guest => "guest",
        }
    }

    pub fn from_str(s: &str) -> Option<Self> {
        match s {
            "admin" => Some(Role::Admin),
            "user" => Some(Role::User),
            "guest" => Some(Role::Guest),
            _ => None,
        }
    }
}

use crate::auth::{AuthError, Identity, Role, TokenClaims, AuthToken};
use jsonwebtoken::{decode, encode, DecodingKey, EncodingKey, Header, Validation};
use chrono::Utc;

/// Token service for JWT operations
pub struct TokenService {
    secret: String,
    token_ttl: i64, // seconds
}

impl TokenService {
    pub fn new(secret: String, token_ttl: i64) -> Self {
        TokenService { secret, token_ttl }
    }

    /// Generate JWT token for identity
    pub fn generate(&self, identity: &Identity) -> Result<AuthToken, AuthError> {
        let now = Utc::now().timestamp();
        let exp = now + self.token_ttl;

        let claims = TokenClaims {
            sub: identity.user_id.clone(),
            username: identity.username.clone(),
            roles: identity
                .roles
                .iter()
                .map(|r| r.as_str().to_string())
                .collect(),
            iat: now,
            exp,
        };

        let encoding_key = EncodingKey::from_secret(self.secret.as_bytes());
        let token = encode(&Header::default(), &claims, &encoding_key)?;

        Ok(AuthToken {
            token,
            token_type: "Bearer".to_string(),
            expires_in: self.token_ttl,
            issued_at: now,
        })
    }

    /// Validate and decode JWT token
    pub fn validate(&self, token: &str) -> Result<Identity, AuthError> {
        let decoding_key = DecodingKey::from_secret(self.secret.as_bytes());

        match decode::<TokenClaims>(token, &decoding_key, &Validation::default()) {
            Ok(data) => {
                let claims = data.claims;

                // Check expiration
                if claims.exp < Utc::now().timestamp() {
                    return Err(AuthError::TokenExpired);
                }

                // Parse roles
                let mut roles = std::collections::HashSet::new();
                for role_str in claims.roles {
                    if let Some(role) = Role::from_str(&role_str) {
                        roles.insert(role);
                    } else {
                        return Err(AuthError::RoleNotFound(role_str));
                    }
                }

                Ok(Identity {
                    user_id: claims.sub,
                    username: claims.username,
                    roles,
                })
            }
            Err(e) => Err(AuthError::JwtError(e)),
        }
    }
}

use crate::auth::{AuthError, Identity, Policy, Role, TokenService};
use std::collections::{HashMap, HashSet};
use std::sync::{Arc, Mutex};

/// In-memory auth state manager
pub struct AuthManager {
    token_service: TokenService,
    policies: Arc<Mutex<HashMap<String, Policy>>>,
    identities: Arc<Mutex<HashMap<String, Identity>>>, // For demo: user_id -> Identity
}

impl AuthManager {
    pub fn new(secret: String, token_ttl: i64) -> Self {
        AuthManager {
            token_service: TokenService::new(secret, token_ttl),
            policies: Arc::new(Mutex::new(HashMap::new())),
            identities: Arc::new(Mutex::new(HashMap::new())),
        }
    }

    /// Register default policies
    pub fn register_default_policies(&self) {
        let mut policies = self.policies.lock().unwrap();

        // Admin policy: full access
        policies.insert(
            "admin_execute".to_string(),
            Policy {
                name: "admin_execute".to_string(),
                resource: "execute".to_string(),
                action: "all".to_string(),
                allowed_roles: {
                    let mut set = HashSet::new();
                    set.insert(Role::Admin);
                    set
                },
            },
        );

        // User policy: can execute and query
        policies.insert(
            "user_execute".to_string(),
            Policy {
                name: "user_execute".to_string(),
                resource: "execute".to_string(),
                action: "read".to_string(),
                allowed_roles: {
                    let mut set = HashSet::new();
                    set.insert(Role::User);
                    set.insert(Role::Admin);
                    set
                },
            },
        );

        // Guest policy: read-only access
        policies.insert(
            "guest_query".to_string(),
            Policy {
                name: "guest_query".to_string(),
                resource: "query".to_string(),
                action: "read".to_string(),
                allowed_roles: {
                    let mut set = HashSet::new();
                    set.insert(Role::Guest);
                    set.insert(Role::User);
                    set.insert(Role::Admin);
                    set
                },
            },
        );
    }

    /// Generate token for identity
    pub fn generate_token(&self, identity: &Identity) -> Result<String, AuthError> {
        let auth_token = self.token_service.generate(identity)?;
        Ok(auth_token.token)
    }

    /// Validate token and return identity
    pub fn validate_token(&self, token: &str) -> Result<Identity, AuthError> {
        self.token_service.validate(token)
    }

    /// Check if identity has permission for resource/action
    pub fn check_permission(
        &self,
        identity: &Identity,
        resource: &str,
        action: &str,
    ) -> Result<bool, AuthError> {
        let policies = self.policies.lock().unwrap();

        let policy_key = format!("{}_{}", identity.roles
            .iter()
            .map(|r| r.as_str())
            .next()
            .unwrap_or("guest"), resource);

        if let Some(policy) = policies.get(&policy_key) {
            Ok(policy.allows(identity))
        } else {
            Ok(false) // No policy = denied
        }
    }

    /// Register a test identity (for demo)
    pub fn register_identity(&self, identity: Identity) {
        let mut identities = self.identities.lock().unwrap();
        identities.insert(identity.user_id.clone(), identity);
    }

    /// Get identity by user_id (for demo)
    pub fn get_identity(&self, user_id: &str) -> Result<Identity, AuthError> {
        let identities = self.identities.lock().unwrap();
        identities
            .get(user_id)
            .cloned()
            .ok_or(AuthError::UserNotFound)
    }
}

use crate::auth::{AuthError, Identity};
use actix_web::{
    dev::{forward_ready, Service, ServiceRequest, ServiceResponse, Transform},
    Error, HttpMessage,
};
use futures::future::LocalBoxFuture;
use std::rc::Rc;

/// Auth extractor middleware - validates Bearer token
pub struct AuthMiddleware {
    secret: String,
}

impl AuthMiddleware {
    pub fn new(secret: String) -> Self {
        AuthMiddleware { secret }
    }
}

impl<S, B> Transform<S, ServiceRequest> for AuthMiddleware
where
    S: Service<ServiceRequest, Response = ServiceResponse<B>, Error = Error> + 'static,
    S::Future: 'static,
    B: 'static,
{
    type Response = ServiceResponse<B>;
    type Error = Error;
    type Transform = AuthMiddlewareService<S>;
    type InitError = ();
    type Future = std::future::Ready<Result<Self::Transform, Self::InitError>>;

    fn new_transform(&self, service: S) -> Self::Future {
        std::future::ready(Ok(AuthMiddlewareService {
            service: Rc::new(service),
            secret: self.secret.clone(),
        }))
    }
}

pub struct AuthMiddlewareService<S> {
    service: Rc<S>,
    secret: String,
}

impl<S, B> Service<ServiceRequest> for AuthMiddlewareService<S>
where
    S: Service<ServiceRequest, Response = ServiceResponse<B>, Error = Error> + 'static,
    S::Future: 'static,
    B: 'static,
{
    type Response = ServiceResponse<B>;
    type Error = Error;
    type Future = LocalBoxFuture<'static, Result<Self::Response, Self::Error>>;

    forward_ready!(service);

    fn call(&self, req: ServiceRequest) -> Self::Future {
        // Extract token from Authorization header
        let auth_header = req.headers().get("Authorization").and_then(|h| h.to_str().ok());

        let identity = match auth_header {
            Some(header) if header.starts_with("Bearer ") => {
                let token = &header[7..];
                // Token will be validated by extractors, just pass through
                Some(token.to_string())
            }
            Some(_) => {
                // Invalid auth header format
                return Box::pin(async move {
                    Err(actix_web::error::ErrorUnauthorized("Invalid auth header"))
                });
            }
            None => None, // No auth header - handler will decide if it's required
        };

        if let Some(token) = identity {
            req.extensions_mut().insert(token);
        }

        let service = self.service.clone();
        Box::pin(async move {
            let res = service.call(req).await?;
            Ok(res)
        })
    }
}

## Overview

The authentication system provides unified, token-based authentication across CLI, HTTP, and WebSocket interfaces. It uses JWT tokens for stateless authentication and implements role-based access control (RBAC) with three roles: Admin, User, and Guest.

### Key Features

- **JWT Token-based Auth**: Stateless, scalable authentication
- **Role-Based Access Control**: Admin, User, Guest roles with granular policies
- **Cross-Interface Integration**: Works seamlessly with CLI (--token flag), HTTP (Bearer token), and WebSocket (token handshake)
- **Policy Enforcement**: Resource/action-based permission checking
- **Optional Auth**: Endpoints support both authenticated and unauthenticated access

## Architecture

### Token Flow

```
1. Client authenticates (via CLI flag, HTTP header, or WebSocket handshake)
2. AuthManager validates credentials/identity
3. TokenService generates JWT token with claims (sub, username, roles, exp)
4. Token returned to client
5. Client includes token in subsequent requests
6. Middleware/Extractors validate token and extract Identity
7. Handlers check permissions via AuthManager.check_permission()
```

### Role Hierarchy

- **Admin**: Full access to all resources and actions
- **User**: Read/write access to execute and query resources
- **Guest**: Read-only access to query resource

### AuthManager API

#### `new(secret: String, token_ttl: i64) -> AuthManager`
Create a new auth manager.

- **Parameters**:
  - `secret`: Secret key for JWT signing (use strong, random value)
  - `token_ttl`: Token time-to-live in seconds

- **Example**:
```rust
let manager = AuthManager::new(
    "your-secret-key-here".to_string(),
    3600 // 1 hour
);
```

#### `generate_token(identity: &Identity) -> Result<String, AuthError>`
Generate JWT token for identity.

- **Returns**: JWT token string or AuthError

- **Example**:
```rust
let identity = Identity {
    user_id: "user123".to_string(),
    username: "john".to_string(),
    roles: {
        let mut set = HashSet::new();
        set.insert(Role::User);
        set
    },
};

let token = manager.generate_token(&identity)?;
```

#### `validate_token(token: &str) -> Result<Identity, AuthError>`
Validate JWT token and extract identity.

- **Parameters**:
  - `token`: JWT token string

- **Returns**: Identity or AuthError (TokenExpired, InvalidToken, RoleNotFound)

- **Example**:
```rust
let identity = manager.validate_token("eyJ0eXAiOiJKV1QiLCJhbGc...")?;
println!("User: {}", identity.username);
```

#### `check_permission(identity: &Identity, resource: &str, action: &str) -> Result<bool, AuthError>`
Check if identity has permission for resource/action.

- **Parameters**:
  - `identity`: User identity
  - `resource`: Resource name (e.g., "execute", "query", "config")
  - `action`: Action name (e.g., "read", "write", "delete")

- **Returns**: Boolean or AuthError

- **Example**:
```rust
if manager.check_permission(&identity, "execute", "write")? {
    // Proceed with write operation
}
```

#### `register_default_policies()`
Register built-in RBAC policies.

- **Registers**:
  - `admin_execute`: Admin can execute any action
  - `user_execute`: User can execute read operations
  - `guest_query`: Guest can query (read-only)

## Error Handling

AuthError variants:

| Error | Code | HTTP Status | Meaning |
|-------|------|-------------|---------|
| InvalidToken | 5001 | 401 | Token format invalid |
| TokenExpired | 5002 | 401 | Token has expired |
| AccessDenied | 5003 | 403 | Insufficient permissions |
| AuthenticationRequired | 5004 | 401 | Auth token required but missing |
| InvalidCredentials | 5005 | 401 | Wrong username/password |
| JwtError | 5006 | 401 | JWT library error |
| UserNotFound | 5007 | 404 | User doesn't exist |
| RoleNotFound | 5008 | 400 | Invalid role in token |

## Security Best Practices

1. **Secret Key Management**: Store JWT secret in environment variable or secure config file
2. **Token TTL**: Use short TTLs (1-2 hours) and implement refresh token flow for long-lived sessions
3. **HTTPS Only**: Always transmit tokens over HTTPS in production
4. **Token Rotation**: Rotate secret keys periodically
5. **Role Principle**: Use principle of least privilege - assign minimum required roles
6. **Audit Logging**: Log all auth events (success and failures)
